# 充值提现系统：链上确认、区块回滚与状态机

## Problem

Crypto 钱包系统需要把链上交易和平台内部账户连接起来。充值可能遇到确认数不足、区块回滚、重复扫描；提现则涉及风控审核、签名、广播、链上确认和失败重试。

## Explanation

充值链路通常从区块扫描开始。扫描服务按高度拉取区块，解析目标地址交易，并根据确认数推进状态。为了应对链重组，系统不能只记录“已扫描到最新高度”，还要保留可回滚窗口，并对交易哈希、地址、金额、币种和区块高度建立唯一约束，保证重复扫描不会重复入账。

提现链路更像一个状态机：用户发起提现后，系统先冻结资产，再进入风控审核，通过后生成待签名任务。签名服务完成签名后广播交易，随后由链上扫描确认交易状态。任何环节失败都不能直接丢弃请求，而是要进入明确的失败、重试或人工处理状态。

## Key Points

- 充值要处理确认数、重复扫描和区块回滚。
- 提现要拆成冻结、审核、签名、广播、确认等状态。
- 热钱包、冷钱包和归集任务要有明确权限边界。
- 所有链上交易都要能从交易哈希追溯到内部业务单号。

## Summary

充值提现系统的关键是把不确定的链上世界，映射成平台内部可追踪、可重试、可审计的状态机。
