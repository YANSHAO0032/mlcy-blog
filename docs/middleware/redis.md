# Redis 缓存设计

## Problem

高并发系统中使用 Redis 时，如何避免缓存穿透、击穿和雪崩？

## Explanation

缓存问题的本质是请求流量和后端承载能力不匹配。缓存穿透通常由不存在的数据反复查询引起，可以通过布隆过滤器或空值缓存缓解；缓存击穿常见于热点 Key 过期瞬间，可以通过互斥锁、逻辑过期或热点永不过期策略处理；缓存雪崩则是大量 Key 同时失效，需要通过过期时间打散、限流降级和多级缓存来降低冲击。

除了命中率，还要关注缓存一致性。对于读多写少场景，常见做法是更新数据库后删除缓存；如果一致性要求高，还需要配合消息队列、重试机制或订阅 binlog 做异步修复。

## Key Points

- 穿透、击穿、雪崩的成因和治理策略不同。
- 热点数据需要单独设计过期与重建机制。
- 删除缓存要考虑并发写和延迟双删等问题。
- 监控命中率、慢查询和内存淘汰策略很关键。

## Summary

Redis 不只是一个高性能 KV 存储，更是系统稳定性的关键缓冲层。缓存方案设计得好，后端系统的抗压能力会有明显提升。
